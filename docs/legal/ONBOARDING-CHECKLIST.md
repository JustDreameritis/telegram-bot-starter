# Client Onboarding Checklist

Everything needed before development begins. Completing this checklist ensures a smooth kickoff and avoids delays.

## Required from Client

### Access & Credentials
- [ ] API keys for all third-party services the project will integrate with
- [ ] Login credentials for any platforms/portals to be automated (use a dedicated service account, not personal)
- [ ] Repository access (if working on an existing codebase)
- [ ] Server/hosting access (if deploying to client infrastructure)
- [ ] Database connection details (if applicable)

### Project Information
- [ ] Detailed requirements document or approved SOW
- [ ] Sample data or test accounts for development
- [ ] Brand guidelines (logos, colors, fonts) if UI work is involved
- [ ] List of stakeholders and their roles (who approves what)
- [ ] Any existing documentation, architecture diagrams, or technical specs

### Communication
- [ ] Preferred communication channel confirmed (Upwork / Slack / Email)
- [ ] Primary point of contact identified
- [ ] Timezone and availability for sync calls (if needed)
- [ ] Feedback turnaround commitment (recommended: 48 hours)

### Timeline
- [ ] Project start date confirmed
- [ ] Key milestones and deadlines reviewed
- [ ] Any blackout dates or holidays that affect the timeline
- [ ] Deployment window preferences (if applicable)

## Provided by Developer

### At Kickoff
- [x] Signed client agreement
- [x] Project timeline with milestones
- [x] Private GitHub repository (or agreed code delivery method)
- [x] Development environment setup
- [x] First progress update within 48 hours of kickoff

### During Development
- [x] Regular progress updates (frequency per SOW)
- [x] Access to staging/preview environment for review
- [x] Prompt responses to client questions (within 24 hours)

---
*Share this checklist with the client at project start. Check off items as they're completed.*
